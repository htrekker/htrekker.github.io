常用算法

GCD(Stein算法)

相比于欧几里得算法，Stein算法减少了乘法计算的次数。仅通过位移和减法运算，能够使计算机更有效的完成计算。

算法描述

  1. 判断输入的两数是否相等，或者任意一个为1。如果满足，则算法结束；
  2. 记输入的两数分别为$A_n$和$B_n$，此时分为以下三种情况：
    • 若$A_n$, $B_n$同时为偶数，则两数同时除以2，并计算结果的最大公约数；
    • 若$A_n$, $B_n$中有且仅有一个偶数，则将为偶数的数字除以2，并计算结果的最大公约数；
    • 若$A_n$, $B_n$同时为奇数，则计算$A_{n+1} = B_n$和$B_{n+1} = \left|A_n - B_n \right|$的最大公约数。

实现

/*
* An implementation of Stein algorithm
*/
public int gcd(int a, int b) {
    if(a == 0 || b == 0) return 0;
    if(a == b) return a;
    
    if((a & 1) == 0 && (b & 1) == 0)
        return gcd(a >> 1, b >> 1) << 1;
    if((b & 1) == 0)
        return gcd(a, b >> 1);
    if((a & 1) == 0)
        return gcd(a >> 1, b);
        
    return gcd(b, Math.abs(a-b));
}

通过最大公约数，我们还可以求出两个数的最小公倍数。记两个数$a$和$b$，两个数的最小公倍数为： $$ \mathrm{lcm}(a, b) = \frac{a \times b}{\mathrm{gcd}(a, b)} $$

public int lcm(int a, int b) {
    return a * b / gcd(a,b);
}

链表是否有环问题(快慢指针)

算法

  1. 设置两个指针，一个每次移动2个节点（快指针），另一个每次移动1个节点（慢指针）；
  2. 每次移动前检查两个指针的情况：
    • 如果两个指针相同，则说明链表有环；
    • 如果快指针为空，或者快指针的下一个节点，即fast.next == null，则链表无环。

实现

public boolean hasCycle(ListNode head) {
    if(head == null) return true;
    ListNode fast = head, slow = head;
    
    while(fast != null && fast.next != null) {
        fast = fast.next.next;
        slow = slow.next;
        if(fast == slow) return true;
    }
    
    return false;
}

这个问题更进一步可以要求我们找出环状链表和单链表的交点。方法如下：

public boolean hasCycle(ListNode head) {
   	...
    ListNode fast = head, slow = head;
    
    ...
        
    ListNode ptr = head;
    while(ptr != slow) {
        ptr = ptr.next;
        slow = slow.next;
    }
    // 此时ptr指向的是交叉节点
    
    ...
}

这种方法的原理是：假设单链表长度为$n$，环状链表长度为$m$。如果此时慢指针移动了$k$次。那么，快指针一定移动了$2k$次。必有$2k = k + c \cdot m$。假设在圈内一共移动了$k^{'}$次，则$k^{'} = c \cdot m - n$。故ptr向前移动$n$次时，慢指针又在圈内移动了$n$次。所以，$k^{'} = c \cdot m - n + n = c \cdot m$。故此时ptr和慢指针均处于环装链表的”入口“，即所需要求的交点。

调度厂算法 (Shunting Yard Algorithm)

调度厂算法 (Shunting Yard Algorithm) 是由Dijkstra提出的将中缀表达式转化为后缀表达式的常用算法。

  • 中缀表达式：与人的计算习惯习惯相同，书面上的计算式即是中缀表达式；
  • 后缀表达式：后缀表达式又称为逆波兰式 (RPN) , 这种表达式最早由Friedrich L. Bauer和Dijkstra在1920年提出。所有运算符均在操作数之后，因此也被称为后缀表达式。由于这种表达式可以有效利用计算机的堆栈结构，从而可以减少计算机访存次数，加快计算速度。

算法

在介绍调度厂算法之前，我们先考虑如何计算后缀表达式。可考虑采用如下算法：

  • 设置一个栈，用来储存当前或者未来将参与计算的操作数；
  • 从后缀表达式的第一项开始运算：
    • 如果当前的操作符为数字，则压入栈内；
    • 如果当前的操作符为运算符，则从栈中弹出该运算所需要个数的操作数进行计算。计算完成后将计算结果压入栈内；
  • 重复上述操作直到到达后缀表达式的末尾，此时栈顶元素为该表达式的运算结果
public int cal(int num1, int num2, int operator) {
    switch(operator) {
        case 0:
            return num1 + num2;
        case 1:
            return num2 - num1;
        case 2:
            return num1 * num2;
        case 3:
            return num2 / num1;
    }

    return 0;
}

/**
* only support four operators (+,-,*,/)
*/
public int evalRPN(String[] tokens) {
    Stack<Integer> numbers = new Stack<>();
    int i = -1;
    while(i++ < tokens.length - 1) {
        switch(tokens[i]) {
            case "+":
                numbers.push(cal(numbers.pop(),numbers.pop(),0));
                break;
            case "-":
                numbers.push(cal(numbers.pop(),numbers.pop(),1));
                break;
            case "*":
                numbers.push(cal(numbers.pop(),numbers.pop(),2));
                break;
            case "/":
                numbers.push(cal(numbers.pop(),numbers.pop(),3));
                break;
            default:
                numbers.push(Integer.valueOf(tokens[i]));
                break;
        }
	}

    return numbers.pop();
}
